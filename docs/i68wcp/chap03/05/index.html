3.5 ターゲット

none

図3.6は4種の抽象化されたターゲットを描いています。前節で紹介したアクションが参照するターゲットは、ユーザが関心を示すデータについての何がしかの側面を意味します。アクション（に与えられた英語の名称）は動詞でしたが、ターゲットには名詞をあてています。SearchアクションとQueryアクションでは、それぞれのターゲットは明確でしょう。一方、Useアクションではターゲットはさほど明確ではないかもしれませんけれど、無縁というわけではありません。たとえば、ユーザが発表する内容や発見を試みる対象物がターゲットにあたります。1

三つの高レベルターゲット（trends, outliers, features）はあらゆるデータに広く該当します。Trendはデータに表れるパターンを高度に特徴づけたものです。2たとえば、上昇、下降、山、谷、停滞などのデータの傾向が該当します。ほとんどのデータにおいて、そういった全体の傾向には合致しないoutlier(外れ値)と呼ばれるものが出現します。3featureはタスクにおいて興味深いなんらかの構造です。ですのでfeatureはタスク依存です。

Attributeは視覚的に表現されるデータの特性です。Attributeのターゲットにおいて、もっとも単純なものは特定の値を見つけることがあります。ほかにも、一定の範囲内の最大値や最小値などのターゲットを見つけることもあります。頻繁に利用される、より高度な視点のターゲットとしてはattributeの値の分布が挙げられます。

依存関係、相関、類似性のように、ターゲットによっては複数のattributeを対象とするものがあります。あるattributeが別のattributeに依存するということは、後者の値が後者の値に依存することを意味します。あるattributeの値が別のattributeの値に関連するとき、これらのあいだに相関があるといいます。ふたつのattributeの類似性 (similarity) は、これらの値について、なんらかの定量尺度を用れば、attributeを類似度に応じて順位づけしたり、違いを論じることができます。

このように傾向、外れ値、分布、相関について把握する抽象タスクはvisを利用する格好の理由となります。これらはドメインに特化して語彙のなかで様々に呼ばれますが、抽象化することで、即座に眺められます。

いくつかのターゲットはデータセットの種類に特化しています。ネットワークデータはノード間の関係をリンクとして定めます。このようなデータにおいて本質的なターゲットは連結関係の構造、つまりネットワークのトポロジーの理解です。トポロジーターゲットには、複数のリンクを介して繋がるノード経路 (path)の理解があります。空間的データについて、ユーザのアクションにしばしば表れるのは、地理的形状の理解と比較でる。
