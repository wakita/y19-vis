解析作業のトリオの三番目にあたるhowは、一連の設計上の選択を経て構成される vis のイディオム (vis idiom) です。図3.7はこのような選択を4つに大分した様子を概説しています。

データをビューのなかに表現して見せる1 一群の系にはデータを空間的に配置するための5つの選択の範囲2 、つまり、値の表現、領域の分割、順列、整列、そして与えられた空間的データの活用があります。 表現系には、空間的ではない視覚チャネル割り当てる手法として、データに対する色、大きさ、角度、形状などの割り当ても考えられます。3 操作系にはビューの変更、ビュー内の要素の選択、ビュー内での視座の変更などがあります。4 視座の変更は大きな選択の幅があるため、それだけで独立した系と見做すこともできるでしょう。 複数のビューにデータの異なる側面を映し出す方法については、複数のビューを併置し、連携する方法、データを複数のビューに分割する方法、そしてビューの上に他のビューを重畳する方法などを選べます。5 The family of how to facet data between views has choices for how to juxtapose and coordinate multiple views, how to partition data between views, and how to superimpose layers on top of each other. 提示されているデータを削減する方法としては、データへのフィルタの適用、たくさんのデータを集約する手法、ひとつのビューに焦点と文脈を埋め込む方法などがあります。6 The family of how to reduce the data shown has the options of filter data away, aggregate many data elements together, and embed focus and context information together within a single view.

この本の残りでは、これらの選択の機会について定義、詳述し議論します。 The rest of this book defines, describes, and discusses these choices in depth.
